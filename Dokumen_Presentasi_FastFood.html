01. Penjelasan Detail Arsitektur Sistem

Struktur arsitektur client-server dan pembagian fungsi frontend & backend.

🖥️ FRONTEND ARCHITECTURE (React + Vite)

Frontend aplikasi dibangun menggunakan pustaka React dengan sistem bundling cepat Vite. Aplikasi dirancang dengan konsep Single Page Application (SPA) yang dinamis, memberikan pengalaman pengguna mobile-first yang mulus layaknya aplikasi native mobile.

Struktur Folder Utama Frontend:

  • 📁 src/components/ : Komponen UI modular reusable (BottomNavbar, FoodCard, SearchBar, LeafletMap).
  • 📁 src/context/ : State global aplikasi (CartContext) untuk sinkronisasi keranjang di semua halaman.
  • 📁 src/lib/ : Klien integrasi layanan eksternal (supabaseClient.js).
  • 📁 src/pages/ : Representasi halaman rute aplikasi (19+ halaman terpisah).
  • 📝 src/App.jsx : Root utama pengatur rute (React Router) & middleware proteksi.

🗄️ BACKEND ARCHITECTURE (Supabase PostgreSQL + External API)

Bagian backend memanfaatkan Supabase sebagai solusi *Backend-as-a-Service (BaaS)* berbasis database relasional PostgreSQL. Selain database awan, backend didukung oleh penyedia data pihak ketiga **TheMealDB API** untuk menyuplai informasi resep makanan riil secara dinamis dan real-time.

Supabase Cloud Database:

  • Supabase Auth: Mengelola registrasi kurir, merchant, customer, dan alur verifikasi email.
  • Tabel Restoran (PostgreSQL): Menyimpan data nama restoran, koordinat geografis (lat, lng), rating, dan estimasi waktu pengiriman.

TheMealDB External API:

  • Integrasi RESTful API: Fetch daftar menu pilihan lewat request HTTP.
  • Pengolahan Data Dinamis: Melakukan mapping data mentah API (nama makanan, asal negara, petunjuk resep) ke format UI internal.

03. Pembahasan Detail 8 Kriteria Penilaian

Penjelasan teknis implementasi fitur berdasarkan komponen penilaian utama.

Kriteria 1 Struktur Proyek & Komponen Modular

Bobot: 15%

Kode sumber diatur dengan pemisahan direktori yang ketat untuk menjaga keterbacaan struktur proyek. Komponen UI dideklarasikan secara modular dalam folder /components. Contoh utama adalah file FoodCard.jsx yang menyediakan tampilan kartu makanan (mendukung format vertikal dan horizontal dengan deteksi kesalahan URL gambar) serta file SearchBar.jsx yang menyediakan kolom input pencarian reusable dengan kontrol reset tombol hapus. Hal ini menghilangkan duplikasi kode tak perlu di berbagai halaman.

Kriteria 2 Props & State (useState)

Bobot: 15%

Komunikasi data dikelola menggunakan mekanisme transfer props dan manajemen status lokal dengan useState. Pada halaman pencarian, input teks diikat pada state searchQuery, sementara pilihan menu diikat pada activeCategory. Di halaman detail makanan, state digunakan untuk mengelola jumlah porsi (quantity), modifikasi pilihan topping/addons (selectedAddons), dan status tombol favorit.

Kriteria 3 Integrasi API (useEffect & Fetch)

Bobot: 15%

Aplikasi terintegrasi dengan **TheMealDB API** secara dinamis menggunakan request asynchronous fetch di dalam hook useEffect. Sistem dilengkapi dengan **Skeleton Loading UI** untuk memberikan umpan balik visual animasi yang premium kepada pengguna saat proses pengambilan data sedang berlangsung. Penanganan kesalahan jaringan disiapkan menggunakan blok `try/catch` untuk memastikan kegagalan koneksi ditangani secara aman dan memunculkan pesan error yang ramah.

Kriteria 4 Styling dengan Tailwind CSS

Bobot: 10%

Seluruh antarmuka dikembangkan dengan **Tailwind CSS v4** menggunakan pendekatan mobile-first design. Desain premium dicapai melalui implementasi gradien warna modern, shadow melayang, batas melengkung melingkar yang halus, serta efek kaca blur (glassmorphism backdrop-blur). Aplikasi juga menyertakan visual *Dark Mode* eksklusif dengan aksen oranye menyala pada portal login admin dan dashboard pengelolaan data admin.

Kriteria 5 Navigasi Multi-Halaman (React Router)

Bobot: 15%

Menggunakan **React Router DOM v7** untuk mengatur alur navigasi aplikasi tanpa memicu pemuatan ulang halaman (SPA). Route dinamis diatur pada path detail makanan /food/:id di mana ID makanan diambil di halaman detail menggunakan hook useParams. Untuk meningkatkan keamanan sistem, ditambahkan juga rute proteksi middleware **ProtectedAdminRoute** yang memvalidasi kunci keamanan sebelum masuk ke dashboard admin, serta rute fallback `*` yang mengarah ke halaman **404 NotFound** kustom.

Kriteria 6 Global State (Context API & Persist)

Bobot: 15%

Keadaan keranjang belanja dikelola sebagai state global melalui **Context API** (CartContext.jsx). State ini membagikan fungsi penambahan, pengurangan, penghapusan, kalkulasi total item (`cartCount`), dan harga pesanan ke semua halaman. Keranjang ini telah ditambahkan **sinkronisasi otomatis ke localStorage** sehingga data belanjaan tetap aman (persisted) meskipun browser dimuat ulang oleh pengguna.

Kriteria 7 Deployment & Handling Error

Bobot: 10%

Aplikasi di-deploy ke awan menggunakan platform serverless **Netlify CDN**. Untuk menyelesaikan masalah umum pemuatan URL secara langsung pada aplikasi SPA (Single Page Application) yang menyebabkan error 404 pada routing, ditambahkan file konfigurasi pengalihan _redirects di direktori publik dengan aturan redirect penuh ke /index.html 200. URL live saat ini adalah: https://fastfood-id.netlify.app.

Kriteria 8 Kebersihan & Keterbacaan Kode

Bobot: 5%

Kode program ditulis mengikuti standar penamaan *camelCase*, bersih, rapi, dan modular. Seluruh file inti (App, CartContext, SearchFilters, Home, dll.) telah dilengkapi dengan dokumentasi komentar berstandar **JSDoc** guna mempermudah pemeliharaan sistem di masa mendatang serta memudahkan pengembang dalam melakukan presentasi kode di hadapan penguji.

Panduan Menyimpan Halaman ini sebagai PDF:

  1. Klik tombol "Cetak PDF Sekarang" di bagian atas layar atau tekan tombol Ctrl + P.
  2. Pada menu cetak browser, ubah tujuan pencetak (*Destination*) menjadi "Save as PDF" atau "Simpan sebagai PDF".
  3. Pastikan opsi "Headers and Footers" dinonaktifkan agar tampilan lebih bersih.
  4. Aktifkan opsi "Background Graphics" agar warna tema oranye dan gradien tetap terlihat indah di PDF Anda.
  5. Simpan file di komputer Anda dan siap dipresentasikan.
